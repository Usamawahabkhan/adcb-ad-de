Convert the SQL Lab into a Step-by-Step Microsoft Fabric Data Pipeline

Goal: Build a Fabric Data Pipeline that runs the warehouse ELT workflow step by step using Script activities.

2. Learning Objectives

  • Convert a manual warehouse SQL lab into an orchestrated Microsoft Fabric Data Pipeline.
  • Use Script activities to run DDL, DML, validation, MERGE, and reporting queries.
  • Keep CREATE VIEW in its own Script activity so it is the first statement in its query batch.
  • Understand how pipeline dependencies control the order of warehouse loading steps.
  • Practice stage, validate, error capture, MERGE, second batch load, aggregate refresh, and final checks.

3. Small Definitions

TermDefinition
Data PipelineA Fabric orchestration item that runs activities in sequence or parallel.
Script activityA pipeline activity used to execute SQL scripts against supported data stores such as Fabric Data Warehouse.
NonQueryScript setting used for commands that create, insert, update, delete, or merge data.
QueryScript setting used when the SQL returns result sets for inspection.
DependencyA link between pipeline activities that controls which activity runs after another activity succeeds.
MERGEAn upsert operation that updates matching target rows and inserts new rows.
Persisted aggregate tableA physical summary table used in this lab as a Warehouse-friendly materialized-view pattern.

4. Pipeline Design

S01_CleanSlate ↓ S02_CreateStageTable ↓ S03_LoadStageBatch6001 ↓ S04_ValidateStage ↓ S05_CaptureLoadErrors ↓ S06_CreateTargetTable ↓ S07_MergeBatch6001 ↓ S08_LoadStageBatch6002 ↓ S09_MergeBatch6002 ↓ S10_CreateSummaryTable ↓ S11_CreateSummaryView ↓ S12_RefreshSummaryTable ↓ S13_FinalChecks
Important: S11_CreateSummaryView must be a separate Script activity because CREATE VIEW must be the first statement in a query batch.

5. Create the Pipeline in Microsoft Fabric

  1. Open the Microsoft Fabric workspace used for the classroom lab.
  2. Select New item and create a Data Pipeline.
  3. Name the pipeline PL_Lab06_Warehouse_ELT.
  4. Add one Script activity for each row in the activity plan below.
  5. For each activity, select the Fabric Warehouse connection in the activity settings.
  6. Use NonQuery for DDL/DML/MERGE steps and Query for validation/final-check steps.
  7. Connect the activities with success dependencies in the order shown.
  8. Save, run, and monitor the pipeline.

Step 4: Validate stage data

Pipeline activity name: S04_ValidateStage

Activity type: Script activity | Script setting: Query

Runs data quality checks. In a real pipeline, you can inspect output in monitoring or split validations into pass/fail control logic.

Code to paste into the Script activity

-- 4A: Row count by batch and source system
SELECT
    BatchID,
    SourceSystem,
    COUNT(*) AS StageRowCount
FROM dbo.stage_RetailSales
GROUP BY BatchID, SourceSystem
ORDER BY BatchID, SourceSystem;

-- 4B: Duplicate business key check
SELECT
    OrderID,
    OrderLineID,
    COUNT(*) AS DuplicateCount
FROM dbo.stage_RetailSales
GROUP BY OrderID, OrderLineID
HAVING COUNT(*) > 1;

-- 4C: Data quality validation report
SELECT
    BatchID,
    OrderID,
    OrderLineID,
    CASE
        WHEN Quantity <= 0 THEN 'Invalid Quantity'
        WHEN UnitPrice <= 0 THEN 'Invalid Unit Price'
        WHEN DiscountPct < 0 OR DiscountPct > 100 THEN 'Invalid Discount Percent'
        WHEN CustomerID <= 0 THEN 'Invalid CustomerID'
        WHEN ProductID <= 0 THEN 'Invalid ProductID'
        WHEN StoreID <= 0 THEN 'Invalid StoreID'
        ELSE 'Valid'
    END AS ValidationStatus
FROM dbo.stage_RetailSales
ORDER BY OrderID, OrderLineID;

Step 5: Create and load error table

Pipeline activity name: S05_CaptureLoadErrors

Activity type: Script activity | Script setting: NonQuery

Creates an error table and captures rejected rows with reasons. This makes bad rows visible instead of silently dropping them.

Code to paste into the Script activity

CREATE TABLE dbo.elt_LoadErrors (
    BatchID           INT           NOT NULL,
    SourceSystem      VARCHAR(50)   NOT NULL,
    OrderID           INT           NOT NULL,
    OrderLineID       INT           NOT NULL,
    ErrorReason       VARCHAR(200)  NOT NULL,
    ErrorCapturedDate DATE          NOT NULL
);

INSERT INTO dbo.elt_LoadErrors
SELECT
    BatchID,
    SourceSystem,
    OrderID,
    OrderLineID,
    CASE
        WHEN Quantity <= 0 THEN 'Invalid Quantity'
        WHEN UnitPrice <= 0 THEN 'Invalid Unit Price'
        WHEN DiscountPct < 0 OR DiscountPct > 100 THEN 'Invalid Discount Percent'
        WHEN CustomerID <= 0 THEN 'Invalid CustomerID'
        WHEN ProductID <= 0 THEN 'Invalid ProductID'
        WHEN StoreID <= 0 THEN 'Invalid StoreID'
        ELSE 'Unknown Error'
    END AS ErrorReason,
    CAST('2024-01-20' AS DATE) AS ErrorCapturedDate
FROM dbo.stage_RetailSales
WHERE Quantity <= 0
   OR UnitPrice <= 0
   OR DiscountPct < 0
   OR DiscountPct > 100
   OR CustomerID <= 0
   OR ProductID <= 0
   OR StoreID <= 0;

Step 7: MERGE valid batch 6001 rows into target

Pipeline activity name: S07_MergeBatch6001

Activity type: Script activity | Script setting: NonQuery

Performs upsert from stage to target. Valid rows are inserted; invalid rows are excluded and already captured in the error table.

Code to paste into the Script activity

MERGE INTO dbo.tgt_RetailSales AS target
USING (
    SELECT
        BatchID,
        SourceSystem,
        OrderID,
        OrderLineID,
        OrderDate,
        CustomerID,
        CustomerName,
        ProductID,
        ProductName,
        CategoryName,
        StoreID,
        StoreName,
        Quantity,
        UnitPrice,
        DiscountPct,
        ROUND(Quantity * UnitPrice, 2) AS GrossSalesAmount,
        ROUND(Quantity * UnitPrice * (DiscountPct / 100), 2) AS DiscountAmount,
        ROUND(Quantity * UnitPrice * (1 - DiscountPct / 100), 2) AS NetSalesAmount,
        LoadDate
    FROM dbo.stage_RetailSales
    WHERE BatchID = 6001
      AND Quantity > 0
      AND UnitPrice > 0
      AND DiscountPct >= 0
      AND DiscountPct <= 100
      AND CustomerID > 0
      AND ProductID > 0
      AND StoreID > 0
) AS source
ON target.OrderID = source.OrderID
AND target.OrderLineID = source.OrderLineID
WHEN MATCHED THEN
    UPDATE SET
        target.OrderDate = source.OrderDate,
        target.CustomerID = source.CustomerID,
        target.CustomerName = source.CustomerName,
        target.ProductID = source.ProductID,
        target.ProductName = source.ProductName,
        target.CategoryName = source.CategoryName,
        target.StoreID = source.StoreID,
        target.StoreName = source.StoreName,
        target.Quantity = source.Quantity,
        target.UnitPrice = source.UnitPrice,
        target.DiscountPct = source.DiscountPct,
        target.GrossSalesAmount = source.GrossSalesAmount,
        target.DiscountAmount = source.DiscountAmount,
        target.NetSalesAmount = source.NetSalesAmount,
        target.SourceSystem = source.SourceSystem,
        target.BatchID = source.BatchID,
        target.LoadDate = source.LoadDate
WHEN NOT MATCHED THEN
    INSERT (
        OrderID,
        OrderLineID,
        OrderDate,
        CustomerID,
        CustomerName,
        ProductID,
        ProductName,
        CategoryName,
        StoreID,
        StoreName,
        Quantity,
        UnitPrice,
        DiscountPct,
        GrossSalesAmount,
        DiscountAmount,
        NetSalesAmount,
        SourceSystem,
        BatchID,
        LoadDate
    )
    VALUES (
        source.OrderID,
        source.OrderLineID,
        source.OrderDate,
        source.CustomerID,
        source.CustomerName,
        source.ProductID,
        source.ProductName,
        source.CategoryName,
        source.StoreID,
        source.StoreName,
        source.Quantity,
        source.UnitPrice,
        source.DiscountPct,
        source.GrossSalesAmount,
        source.DiscountAmount,
        source.NetSalesAmount,
        source.SourceSystem,
        source.BatchID,
        source.LoadDate
    );

Step 10: Create persisted aggregate table

Pipeline activity name: S10_CreateSummaryTable

Activity type: Script activity | Script setting: NonQuery

Creates and loads a persisted aggregate table. This is the Fabric Warehouse-friendly materialized-view pattern.

Code to paste into the Script activity

CREATE TABLE dbo.mv_DailySalesSummary (
    OrderDate        DATE          NOT NULL,
    CategoryName     VARCHAR(100)  NOT NULL,
    StoreName        VARCHAR(200)  NOT NULL,
    TotalOrders      INT           NOT NULL,
    TotalUnits       INT           NOT NULL,
    GrossRevenue     DECIMAL(10,2) NOT NULL,
    TotalDiscount    DECIMAL(10,2) NOT NULL,
    NetRevenue       DECIMAL(10,2) NOT NULL
);

INSERT INTO dbo.mv_DailySalesSummary
SELECT
    OrderDate,
    CategoryName,
    StoreName,
    COUNT(DISTINCT OrderID) AS TotalOrders,
    SUM(Quantity) AS TotalUnits,
    SUM(GrossSalesAmount) AS GrossRevenue,
    SUM(DiscountAmount) AS TotalDiscount,
    SUM(NetSalesAmount) AS NetRevenue
FROM dbo.tgt_RetailSales
GROUP BY OrderDate, CategoryName, StoreName;

Step 11: Create standard view on the summary table

Pipeline activity name: S11_CreateSummaryView

Activity type: Script activity | Script setting: NonQuery

Creates the reporting view. Keep this as its own Script activity because CREATE VIEW must be the first statement in a query batch.

Code to paste into the Script activity

CREATE VIEW dbo.vw_DailySalesSummary AS
SELECT
    OrderDate,
    CategoryName,
    StoreName,
    TotalOrders,
    TotalUnits,
    GrossRevenue,
    TotalDiscount,
    NetRevenue
FROM dbo.mv_DailySalesSummary;

Step 12: Refresh persisted aggregate table

Pipeline activity name: S12_RefreshSummaryTable

Activity type: Script activity | Script setting: NonQuery

Refreshes the persisted aggregate after curated target data changes.

Code to paste into the Script activity

DELETE FROM dbo.mv_DailySalesSummary;

INSERT INTO dbo.mv_DailySalesSummary
SELECT
    OrderDate,
    CategoryName,
    StoreName,
    COUNT(DISTINCT OrderID) AS TotalOrders,
    SUM(Quantity) AS TotalUnits,
    SUM(GrossSalesAmount) AS GrossRevenue,
    SUM(DiscountAmount) AS TotalDiscount,
    SUM(NetSalesAmount) AS NetRevenue
FROM dbo.tgt_RetailSales
GROUP BY OrderDate, CategoryName, StoreName;

Teacher Notes and Common Issues

  • If the view creation fails with CREATE VIEW must be the first statement in a query batch, confirm that S11_CreateSummaryView contains only the CREATE VIEW statement.
  • If rerunning the pipeline fails on existing objects, confirm S01_CleanSlate ran successfully.
  • If no invalid rows appear in elt_LoadErrors, confirm batch 6001 sample data includes the invalid quantity, price, and discount rows.
  • If summary values appear stale, rerun S12_RefreshSummaryTable.
