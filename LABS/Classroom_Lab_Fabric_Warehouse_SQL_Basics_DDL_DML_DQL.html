Course Overview

This HTML guide contains five classroom labs plus an additional DDL/DML/DQL loading-pattern section. The labs use the Microsoft Fabric Warehouse sample objects such as dbo.Trip, dbo.Date, and dbo.Time, and the loading-pattern section uses safe lab-owned tables prefixed with lab_.

Important: Run DML examples only against the lab-owned tables created in this guide. Avoid updating or deleting rows in Fabric sample tables.

Lab 1: Set Up Microsoft Fabric Workspace and Create a Warehouse Sample

FieldValue
Lab number1
Estimated duration30–40 minutes
ModalityHands-on environment setup
Primary Fabric itemsWorkspace, Warehouse

Learning objectives

  • Create or access a Fabric-enabled workspace.
  • Create a Warehouse sample in Fabric and verify that sample data is loaded.
  • Understand the role of the Warehouse sample as a safe practice environment for SQL and analytics labs.

Concepts covered

  • Fabric workspace
  • Warehouse item
  • Sample data initialization
  • Warehouse object browser
  • T-SQL endpoint awareness

Detailed steps

  1. Sign in to the Microsoft Fabric portal.
  2. Navigate to Workspaces and create a new Fabric-enabled workspace if one does not already exist.
  3. Select New and create a new Warehouse sample. Give it a unique descriptive name.
  4. Wait until Fabric finishes creating the Warehouse and loading the sample data.
  5. Open the Warehouse and review the Explorer pane to locate schemas, tables, and views.
  6. Open a new SQL query window and run the validation query.

SQL validation query

SELECT
    s.name  AS schema_name,
    o.name  AS object_name,
    o.type_desc AS object_type
FROM sys.objects o
JOIN sys.schemas s
    ON o.schema_id = s.schema_id
WHERE o.type IN ('U','V')
ORDER BY s.name, o.type_desc, o.name;

Expected results

  • The Warehouse opens successfully.
  • The object browser shows sample tables and views.
  • The SQL query returns a list of relational objects available in the Warehouse.

Extension activities

  • Record the names of key sample tables that seem fact-like versus dimension-like.
  • Identify one object you expect to use later for joins and one for aggregations.

Troubleshooting

  • If the Warehouse sample does not appear, verify workspace capacity and Fabric enablement.
  • If the query window does not run, confirm that the Warehouse item itself is open and not a Lakehouse SQL endpoint.
Production note: In real projects, warehouse provisioning includes naming standards, access controls, dataset ownership, and environment separation such as development, test, and production.

Lab 2: Explore the Warehouse Sample and Profile Key Tables

FieldValue
Lab number2

Primary Fabric itemsWarehouse, SQL Query Editor

Learning objectives

  • Profile important sample tables and identify candidate business entities.
  • Estimate row volumes and discover useful columns for filtering, joining, and aggregation.
  • Practice source profiling before writing analytic logic.

Suggested profiling tasks

  1. Select three core sample tables and inspect the first 100 rows of each.
  2. Count total rows per table.
  3. List distinct values for one candidate category or status column.
  4. Profile nulls on one critical business column.
  5. Identify probable primary or business keys.

Example profiling queries

SELECT TOP 100 * FROM dbo.Trip;
SELECT TOP 100 * FROM dbo.Date;
SELECT TOP 100 * FROM dbo.Time;

SELECT 'Trip' AS table_name, COUNT(*) AS row_count FROM dbo.Trip
UNION ALL
SELECT 'Date', COUNT(*) FROM dbo.Date
UNION ALL
SELECT 'Time', COUNT(*) FROM dbo.Time;

SELECT PassengerCount, COUNT(*) AS trip_count
FROM dbo.Trip
GROUP BY PassengerCount
ORDER BY PassengerCount;

-- Null profile example
SELECT
    SUM(CASE WHEN FareAmount IS NULL THEN 1 ELSE 0 END) AS null_fare_amount_rows,
    SUM(CASE WHEN TripDistanceMiles IS NULL THEN 1 ELSE 0 END) AS null_distance_rows,
    COUNT(*) AS total_rows
FROM dbo.Trip;

Deliverable

Create a short object profiling note that lists each profiled table, likely business purpose, candidate join key, row count, and one potential risk or quality issue.

Lab 5: Advanced SQL on the Warehouse Sample

FieldValue
Lab number5
Estimated duration75–90 minutes
ModalityHands-on advanced SQL lab
Primary Fabric itemsWarehouse, SQL Query Editor

Learning objectives

  • Join sample tables to combine measures with contextual dimensions.
  • Use CTEs to structure complex logic.
  • Apply window functions for running totals, ranking, and row-level analytics.
  • Generate multi-level summaries with grouping sets.

Part A: Joins

SELECT TOP 100
    d.Date,
    tm.HourNumber,
    tm.MinuteNumber,
    t.PassengerCount,
    t.TripDistanceMiles,
    t.FareAmount
FROM dbo.Trip AS t
JOIN dbo.Date AS d
    ON t.DateID = d.DateID
JOIN dbo.Time AS tm
    ON t.PickupTimeID = tm.TimeID;

Step-by-step breakdown

  1. SELECT TOP 100 returns the first 100 rows for quick analysis.
  2. FROM dbo.Trip AS t uses the main fact-like trip table.
  3. JOIN dbo.Date AS d adds calendar context through DateID.
  4. JOIN dbo.Time AS tm adds time context through PickupTimeID.
  5. The selected columns provide date, time, passenger count, distance, and fare insights.

Part B: CTE and ranking

WITH hourly_revenue AS (
    SELECT
        tm.HourlyBucket,
        SUM(t.FareAmount + t.TipAmount) AS total_revenue
    FROM dbo.Trip AS t
    JOIN dbo.Time AS tm
        ON t.PickupTimeID = tm.TimeID
    GROUP BY tm.HourlyBucket
)
SELECT
    HourlyBucket,
    total_revenue,
    DENSE_RANK() OVER (ORDER BY total_revenue DESC) AS revenue_rank
FROM hourly_revenue
ORDER BY revenue_rank;

Part C: Grouping sets

SELECT
    d.IsWeekday,
    tm.HourlyBucket,
    COUNT(*) AS trip_count
FROM dbo.Trip AS t
JOIN dbo.Date AS d
    ON t.DateID = d.DateID
JOIN dbo.Time AS tm
    ON t.PickupTimeID = tm.TimeID
GROUP BY GROUPING SETS (
    (d.IsWeekday, tm.HourlyBucket),
    (d.IsWeekday),
    ()
);

Part D: Running totals

WITH daily_fare AS (
    SELECT
        d.[Date] AS CalendarDate,
        SUM(t.FareAmount) AS daily_fare
    FROM dbo.Trip AS t
    JOIN dbo.[Date] AS d
        ON t.DateID = d.DateID
    GROUP BY d.[Date]
)
SELECT
    CalendarDate,
    daily_fare,
    SUM(daily_fare) OVER (
        ORDER BY CalendarDate
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS running_total_fare
FROM daily_fare
ORDER BY CalendarDate;

Learner deliverable

Write one original business question and answer it using at least one join, one CTE, and one window function.

DDL & Warehouse Loading Patterns: DDL, DML, and DQL Examples

Purpose: This section gives learners safe classroom tables for practicing DDL, DML, DQL, and warehouse loading patterns without changing the Fabric sample tables.

Key terms

  • DDL: Data Definition Language, such as CREATE TABLE and DROP TABLE.
  • DML: Data Manipulation Language, such as INSERT, UPDATE, DELETE, and loading patterns.
  • DQL: Data Query Language, mainly SELECT queries used to read and analyze data.
  • Warehouse loading pattern: Load raw data into a stage table, validate it, capture errors, then insert valid rows into curated tables.
-- ============================================================
-- DDL & WAREHOUSE LOADING PATTERNS
-- Safe classroom tables. These do not modify Fabric sample tables.
-- ============================================================

DROP TABLE IF EXISTS dbo.lab_LoadErrors;
DROP TABLE IF EXISTS dbo.lab_FactSales;
DROP TABLE IF EXISTS dbo.lab_StageSales;
DROP TABLE IF EXISTS dbo.lab_Product;
DROP TABLE IF EXISTS dbo.lab_Customer;

-- DDL: Create dimension-like lookup tables
CREATE TABLE dbo.lab_Customer (
    CustomerID   INT          NOT NULL,
    CustomerName VARCHAR(200) NOT NULL,
    City         VARCHAR(100) NOT NULL,
    LoyaltyTier  VARCHAR(50)  NOT NULL
);

CREATE TABLE dbo.lab_Product (
    ProductID    INT           NOT NULL,
    ProductName  VARCHAR(200)  NOT NULL,
    CategoryName VARCHAR(100)  NOT NULL,
    UnitPrice    DECIMAL(10,2) NOT NULL
);

-- DDL: Create raw stage table
-- Stage columns are often flexible because file/API data can arrive as text.
CREATE TABLE dbo.lab_StageSales (
    BatchID      INT           NOT NULL,
    OrderID      INT           NOT NULL,
    OrderDate    VARCHAR(20)   NOT NULL,
    CustomerID   INT           NOT NULL,
    ProductID    INT           NOT NULL,
    Quantity     INT           NOT NULL,
    UnitPrice    DECIMAL(10,2) NOT NULL,
    DiscountPct  DECIMAL(5,2)  NOT NULL
);

-- DDL: Create curated fact table
CREATE TABLE dbo.lab_FactSales (
    SalesKey         INT           NOT NULL,
    OrderID          INT           NOT NULL,
    OrderDate        DATE          NOT NULL,
    CustomerID       INT           NOT NULL,
    ProductID        INT           NOT NULL,
    Quantity         INT           NOT NULL,
    UnitPrice        DECIMAL(10,2) NOT NULL,
    DiscountPct      DECIMAL(5,2)  NOT NULL,
    GrossSalesAmount DECIMAL(10,2) NOT NULL,
    DiscountAmount   DECIMAL(10,2) NOT NULL,
    NetSalesAmount   DECIMAL(10,2) NOT NULL
);

CREATE TABLE dbo.lab_LoadErrors (
    BatchID      INT          NOT NULL,
    OrderID      INT          NOT NULL,
    ErrorReason  VARCHAR(200) NOT NULL
);

-- DML: Insert sample lookup data
INSERT INTO dbo.lab_Customer VALUES
(1, 'James Wilson', 'New York', 'Gold'),
(2, 'Sophia Martinez', 'Los Angeles', 'Platinum'),
(3, 'Ethan Brown', 'Chicago', 'Silver');

INSERT INTO dbo.lab_Product VALUES
(101, 'Wireless Headphones', 'Electronics', 89.99),
(102, 'Smart Watch', 'Electronics', 249.99),
(103, 'Running Jacket', 'Clothing', 79.99);

-- DML: Load raw stage data
-- Includes one invalid row with negative quantity.
INSERT INTO dbo.lab_StageSales VALUES
(9001, 5001, '2024-04-01', 1, 101, 2,  89.99,  0.00),
(9001, 5002, '2024-04-02', 2, 102, 1, 249.99, 10.00),
(9001, 5003, '2024-04-03', 3, 103, -1, 79.99,  5.00);

-- DQL: Inspect staged rows
SELECT *
FROM dbo.lab_StageSales
ORDER BY OrderID;

-- DQL: Validate stage data
SELECT
    BatchID,
    OrderID,
    CASE
        WHEN TRY_CAST(OrderDate AS DATE) IS NULL THEN 'Invalid OrderDate'
        WHEN Quantity <= 0 THEN 'Invalid Quantity'
        WHEN UnitPrice <= 0 THEN 'Invalid Unit Price'
        WHEN DiscountPct < 0 OR DiscountPct > 100 THEN 'Invalid Discount Percent'
        ELSE 'Valid'
    END AS ValidationStatus
FROM dbo.lab_StageSales
ORDER BY OrderID;

-- DML loading pattern: Capture rejected rows
INSERT INTO dbo.lab_LoadErrors
SELECT
    BatchID,
    OrderID,
    CASE
        WHEN TRY_CAST(OrderDate AS DATE) IS NULL THEN 'Invalid OrderDate'
        WHEN Quantity <= 0 THEN 'Invalid Quantity'
        WHEN UnitPrice <= 0 THEN 'Invalid Unit Price'
        WHEN DiscountPct < 0 OR DiscountPct > 100 THEN 'Invalid Discount Percent'
        ELSE 'Unknown Error'
    END AS ErrorReason
FROM dbo.lab_StageSales
WHERE TRY_CAST(OrderDate AS DATE) IS NULL
   OR Quantity <= 0
   OR UnitPrice <= 0
   OR DiscountPct < 0
   OR DiscountPct > 100;

-- DML loading pattern: Insert valid rows into curated fact table
INSERT INTO dbo.lab_FactSales
SELECT
    ROW_NUMBER() OVER (ORDER BY s.OrderID) AS SalesKey,
    s.OrderID,
    TRY_CAST(s.OrderDate AS DATE) AS OrderDate,
    s.CustomerID,
    s.ProductID,
    s.Quantity,
    s.UnitPrice,
    s.DiscountPct,
    ROUND(s.Quantity * s.UnitPrice, 2) AS GrossSalesAmount,
    ROUND(s.Quantity * s.UnitPrice * (s.DiscountPct / 100), 2) AS DiscountAmount,
    ROUND(s.Quantity * s.UnitPrice * (1 - s.DiscountPct / 100), 2) AS NetSalesAmount
FROM dbo.lab_StageSales AS s
WHERE TRY_CAST(s.OrderDate AS DATE) IS NOT NULL
  AND s.Quantity > 0
  AND s.UnitPrice > 0
  AND s.DiscountPct >= 0
  AND s.DiscountPct <= 100;

-- DQL: Query curated output with joins
SELECT
    f.OrderID,
    f.OrderDate,
    c.CustomerName,
    c.LoyaltyTier,
    p.ProductName,
    p.CategoryName,
    f.Quantity,
    f.NetSalesAmount
FROM dbo.lab_FactSales AS f
JOIN dbo.lab_Customer AS c
    ON f.CustomerID = c.CustomerID
JOIN dbo.lab_Product AS p
    ON f.ProductID = p.ProductID
ORDER BY f.OrderDate;

-- DQL: Aggregate curated output
SELECT
    p.CategoryName,
    COUNT(*) AS sales_rows,
    SUM(f.Quantity) AS units_sold,
    SUM(f.NetSalesAmount) AS net_revenue
FROM dbo.lab_FactSales AS f
JOIN dbo.lab_Product AS p
    ON f.ProductID = p.ProductID
GROUP BY p.CategoryName
ORDER BY net_revenue DESC;

-- DML: Update example on lab table only
UPDATE dbo.lab_Customer
SET LoyaltyTier = 'Platinum'
WHERE CustomerID = 1;

-- DML: Delete example on error table only
DELETE FROM dbo.lab_LoadErrors
WHERE ErrorReason = 'Unknown Error';

-- Final check
SELECT * FROM dbo.lab_LoadErrors ORDER BY OrderID;
SELECT * FROM dbo.lab_FactSales ORDER BY OrderID;
