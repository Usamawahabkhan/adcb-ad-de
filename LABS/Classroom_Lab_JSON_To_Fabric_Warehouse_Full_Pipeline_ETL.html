2. Learning Objectives

  • Understand two ways to load data into a Fabric Warehouse: SQL script and Copy activity.
  • Create a warehouse stage table for JSON-shaped retail sales data.
  • Use Fabric Pipeline activities to orchestrate loading and transformation.
  • Validate staged data before loading curated tables.
  • Capture invalid records in an error table.
  • Use MERGE to load valid data into a curated warehouse table.
  • Create a Gold summary table and a reporting view for analytics.

3. Small Definitions

TermSimple definition
JSONA semi-structured text format commonly used by APIs and applications.
Copy activityA Fabric Pipeline activity used to move data from a source to a destination.
Script activityA Fabric Pipeline activity used to run SQL against Fabric Warehouse.
Stage tableA landing table where raw or incoming rows are loaded before validation.
ValidationSQL checks that identify bad rows before rows enter curated output.
Curated tableA clean table that stores validated and transformed business data.
Gold tableA reporting-ready summary table used by analysts or Power BI.
MERGEA SQL upsert operation that inserts new rows and updates existing rows.

5. End-to-End Pipeline Architecture

JSON file: retail_sales_sample.json | | Option A: Fabric Pipeline Copy activity | Option B: SQL INSERT script for classroom fallback v Warehouse stage table: dbo.stage_RetailSalesJson | v Validation SQL | +---- invalid rows ----> dbo.elt_LoadErrors | v MERGE valid rows | v Curated table: dbo.curated_RetailSales | v Gold table: dbo.gold_DailySalesSummary | v Reporting view: dbo.vw_DailySalesSummary

6. Full Fabric Pipeline Experience

  1. Create or open a Fabric Warehouse.
  2. Create a new Fabric Data Pipeline named PL_JSON_To_Warehouse_ETL.
  3. Add Script activity A01_CleanSlate and paste Step 1 SQL.
  4. Add Script activity A02_CreateStageTable and paste Step 2 SQL.
  5. Add Copy activity A03_CopyJsonToStage to load the JSON file into dbo.stage_RetailSalesJson.
  6. If Copy activity is not available in the classroom environment, use the SQL-only fallback in Step 3 Option B.
  7. Add Script activity A04_ValidateStage and set the script type to Query.
  8. Add Script activity A05_CaptureErrors to store rejected rows.
  9. Add Script activity A06_CreateTargetAndMerge to create the curated table and MERGE valid rows.
  10. Add Script activity A07_CreateGoldSummary to build the Gold summary table.
  11. Add Script activity A08_CreateReportingView. Keep CREATE VIEW alone in this activity.
  12. Add Script activity A09_FinalChecks and set the script type to Query.
Important: Keep CREATE VIEW in a separate Script activity because CREATE VIEW must be the first statement in its query batch.

8. Copy Activity Configuration

Source

  • Source type: JSON file from Lakehouse Files, OneLake, or another supported file source.
  • File name: retail_sales_sample.json
  • Format: Array of JSON objects.

Destination

  • Destination type: Fabric Data Warehouse

Mapping

Map each JSON property to the matching warehouse column. A compact mapping example is included in Step 3 Option A.

Step 4 SQL: Validate staged data

SELECT
    BatchID,
    COUNT(*) AS StageRows,
    SUM(CASE WHEN Quantity <= 0 THEN 1 ELSE 0 END) AS InvalidQuantityRows,
    SUM(CASE WHEN UnitPrice <= 0 THEN 1 ELSE 0 END) AS InvalidPriceRows,
    SUM(CASE WHEN DiscountPct < 0 OR DiscountPct > 100 THEN 1 ELSE 0 END) AS InvalidDiscountRows
FROM dbo.stage_RetailSalesJson
GROUP BY BatchID;

SELECT
    OrderID,
    OrderLineID,
    CASE
        WHEN Quantity <= 0 THEN 'Invalid Quantity'
        WHEN UnitPrice <= 0 THEN 'Invalid Unit Price'
        WHEN DiscountPct < 0 OR DiscountPct > 100 THEN 'Invalid Discount Percent'
        ELSE 'Valid'
    END AS ValidationStatus
FROM dbo.stage_RetailSalesJson
ORDER BY OrderID, OrderLineID;

Step 5 SQL: Capture rejected rows

CREATE TABLE dbo.elt_LoadErrors (
    BatchID           INT          NOT NULL,
    SourceSystem      VARCHAR(50)  NOT NULL,
    OrderID           INT          NOT NULL,
    OrderLineID       INT          NOT NULL,
    ErrorReason       VARCHAR(200) NOT NULL,
    ErrorCapturedDate DATE         NOT NULL
);

INSERT INTO dbo.elt_LoadErrors
SELECT
    BatchID,
    SourceSystem,
    OrderID,
    OrderLineID,
    CASE
        WHEN Quantity <= 0 THEN 'Invalid Quantity'
        WHEN UnitPrice <= 0 THEN 'Invalid Unit Price'
        WHEN DiscountPct < 0 OR DiscountPct > 100 THEN 'Invalid Discount Percent'
        ELSE 'Unknown Error'
    END AS ErrorReason,
    CAST('2024-02-05' AS DATE) AS ErrorCapturedDate
FROM dbo.stage_RetailSalesJson
WHERE Quantity <= 0
   OR UnitPrice <= 0
   OR DiscountPct < 0
   OR DiscountPct > 100;

Step 6 SQL: Create target and MERGE valid rows

CREATE TABLE dbo.curated_RetailSales (
    OrderID          INT           NOT NULL,
    OrderLineID      INT           NOT NULL,
    OrderDate        DATE          NOT NULL,
    CustomerID       INT           NOT NULL,
    CustomerName     VARCHAR(200)  NOT NULL,
    ProductID        INT           NOT NULL,
    ProductName      VARCHAR(200)  NOT NULL,
    CategoryName     VARCHAR(100)  NOT NULL,
    StoreID          INT           NOT NULL,
    StoreName        VARCHAR(200)  NOT NULL,
    Quantity         INT           NOT NULL,
    UnitPrice        DECIMAL(10,2) NOT NULL,
    DiscountPct      DECIMAL(5,2)  NOT NULL,
    GrossSalesAmount DECIMAL(10,2) NOT NULL,
    DiscountAmount   DECIMAL(10,2) NOT NULL,
    NetSalesAmount   DECIMAL(10,2) NOT NULL,
    SourceSystem     VARCHAR(50)   NOT NULL,
    BatchID          INT           NOT NULL,
    LoadDate         DATE          NOT NULL
);

MERGE INTO dbo.curated_RetailSales AS target
USING (
    SELECT
        BatchID,
        SourceSystem,
        OrderID,
        OrderLineID,
        OrderDate,
        CustomerID,
        CustomerName,
        ProductID,
        ProductName,
        CategoryName,
        StoreID,
        StoreName,
        Quantity,
        UnitPrice,
        DiscountPct,
        ROUND(Quantity * UnitPrice, 2) AS GrossSalesAmount,
        ROUND(Quantity * UnitPrice * (DiscountPct / 100), 2) AS DiscountAmount,
        ROUND(Quantity * UnitPrice * (1 - DiscountPct / 100), 2) AS NetSalesAmount,
        LoadDate
    FROM dbo.stage_RetailSalesJson
    WHERE Quantity > 0
      AND UnitPrice > 0
      AND DiscountPct >= 0
      AND DiscountPct <= 100
) AS source
ON target.OrderID = source.OrderID
AND target.OrderLineID = source.OrderLineID
WHEN MATCHED THEN
    UPDATE SET
        target.Quantity = source.Quantity,
        target.UnitPrice = source.UnitPrice,
        target.DiscountPct = source.DiscountPct,
        target.GrossSalesAmount = source.GrossSalesAmount,
        target.DiscountAmount = source.DiscountAmount,
        target.NetSalesAmount = source.NetSalesAmount,
        target.SourceSystem = source.SourceSystem,
        target.BatchID = source.BatchID,
        target.LoadDate = source.LoadDate
WHEN NOT MATCHED THEN
    INSERT (
        OrderID, OrderLineID, OrderDate, CustomerID, CustomerName,
        ProductID, ProductName, CategoryName, StoreID, StoreName,
        Quantity, UnitPrice, DiscountPct, GrossSalesAmount,
        DiscountAmount, NetSalesAmount, SourceSystem, BatchID, LoadDate
    )
    VALUES (
        source.OrderID, source.OrderLineID, source.OrderDate, source.CustomerID, source.CustomerName,
        source.ProductID, source.ProductName, source.CategoryName, source.StoreID, source.StoreName,
        source.Quantity, source.UnitPrice, source.DiscountPct, source.GrossSalesAmount,
        source.DiscountAmount, source.NetSalesAmount, source.SourceSystem, source.BatchID, source.LoadDate
    );

Step 7 SQL: Create Gold summary table

CREATE TABLE dbo.gold_DailySalesSummary (
    OrderDate      DATE          NOT NULL,
    CategoryName   VARCHAR(100)  NOT NULL,
    StoreName      VARCHAR(200)  NOT NULL,
    TotalOrders    INT           NOT NULL,
    TotalUnits     INT           NOT NULL,
    GrossRevenue   DECIMAL(10,2) NOT NULL,
    TotalDiscount  DECIMAL(10,2) NOT NULL,
    NetRevenue     DECIMAL(10,2) NOT NULL
);

INSERT INTO dbo.gold_DailySalesSummary
SELECT
    OrderDate,
    CategoryName,
    StoreName,
    COUNT(DISTINCT OrderID) AS TotalOrders,
    SUM(Quantity) AS TotalUnits,
    SUM(GrossSalesAmount) AS GrossRevenue,
    SUM(DiscountAmount) AS TotalDiscount,
    SUM(NetSalesAmount) AS NetRevenue
FROM dbo.curated_RetailSales
GROUP BY OrderDate, CategoryName, StoreName;

Step 8 SQL: Create reporting view as separate activity

CREATE VIEW dbo.vw_DailySalesSummary AS
SELECT
    OrderDate,
    CategoryName,
    StoreName,
    TotalOrders,
    TotalUnits,
    GrossRevenue,
    TotalDiscount,
    NetRevenue
FROM dbo.gold_DailySalesSummary;

Step 9 SQL: Final checks

SELECT * FROM dbo.elt_LoadErrors ORDER BY OrderID, OrderLineID;
SELECT * FROM dbo.curated_RetailSales ORDER BY OrderID, OrderLineID;
SELECT * FROM dbo.vw_DailySalesSummary ORDER BY OrderDate, CategoryName, StoreName;

Teacher Notes and Common Issues

  • If JSON copy is difficult in class, run the SQL-only manual load section instead.
  • If CREATE VIEW fails, make sure the view creation is in a separate Script activity.
  • If invalid rows are not captured, verify that the sample JSON includes the invalid quantity row.
  • If the summary view returns no rows, verify that the MERGE step inserted valid rows into dbo.curated_RetailSales.
  • For repeat runs, execute the cleanup step first.
